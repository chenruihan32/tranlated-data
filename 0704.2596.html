Abstract

This paper deals with the problem of increasing the minimum distance of a linear code by adding one or more columns to the generator matrix. Several methods to compute extensions of linear codes are presented. Many codes improving the previously known lower bounds on the minimum distance have been found.

I Introduction

In this paper we consider the question when a linear code C=[n,k,d]q𝐶subscript𝑛𝑘𝑑𝑞C=[n,k,d]_{q} over 𝔽qsubscript𝔽𝑞{\mathbb{F}}_{q} of length n𝑛n, dimension k𝑘k, and minimum distance d𝑑d can be extended to a code C′=[n+1,k,d+1]qsuperscript𝐶′subscript𝑛1𝑘𝑑1𝑞C^{\prime}=[n+1,k,d+1]_{q}. It is a well known fact in coding theory that every binary linear code C=[n,k,d]2𝐶subscript𝑛𝑘𝑑2C=[n,k,d]_{2} whose minimum weight d𝑑d is odd can be extended to a code [n+1,k,d+1]2subscript𝑛1𝑘𝑑12[n+1,k,d+1]_{2} by adding a single parity check. This can also be expressed in terms of Construction X [17] applied to the code C𝐶C, its one-codimensional even-weight subcode C0subscript𝐶0C_{0}, and the trivial code [1,1,1]2subscript1112[1,1,1]_{2}. While this result does not have an immediate generalization to non-binary alphabets, Hill and Lizak [9, 10] proved the following theorem:

Theorem 1

Let C𝐶C be an [n,k,d]qsubscript𝑛𝑘𝑑𝑞[n,k,d]_{q} code with gcd⁡(d,q)=1𝑑𝑞1\gcd(d,q)=1 and with all weights congruent to 00 or d𝑑d (modulo q𝑞q). Then C𝐶C can be extended to an [n+1,k,d+1]qsubscript𝑛1𝑘𝑑1𝑞[n+1,k,d+1]_{q} code all of whose weights are congruent to 00 or d+1𝑑1d+1 (modulo q𝑞q).

In order to apply this theorem, knowledge about the weight spectrum of the code C𝐶C is required. A generalization of this theorem due to Simonis [16] can be applied when additionally information on the weight distribution of the code C𝐶C is available. The special cases with gcd⁡(q,d)=1𝑞𝑑1\gcd(q,d)=1 and in particular ternary codes have been treated by Maruta [13, 14, 15]. However, these results are of rather theoretical nature and have mainly be used to prove the non-existence of codes with certain parameters. The application to a specific code might be difficult since one has to compute information on the weight distribution of the code first.

II-B Computing the minimum weight codewords

In the sequel we describe an algorithm to compute the minimum distance of a code as well as all words of minimum weight. The algorithm is based on an algorithm by Zimmermann to compute the minimum distance (see [19] and [1, Algorithmus 1.3.6]) which improved an algorithm by Brouwer. Together with some further improvements, the algorithm is implemented in the computer algebra system MAGMA (see [2, 8]).

The main idea of the algorithm is to enumerate the codewords in such a way that one does not only obtain an upper bound on the minimum distance of the code via the minimum of the weight of the words that have been encountered, but to establish lower bounds on the minimum distance as well. For this, we are using a collection of systematic generator matrices Gjsubscript𝐺𝑗G_{j} with corresponding information sets ℐjsubscriptℐ𝑗{\cal I}_{j}. III Computing Extensions

III-A Exhaustive search

Given the set 𝒥dsubscript𝒥𝑑{\cal J}_{d} of information vectors of the minimum weight codewords, one can use an exhaustive search to find a column vector 𝒙𝒙\textstyle x or a matrix X𝑋X that fulfills condition (2) or (1). In total there are qm​k−1superscript𝑞𝑚𝑘1q^{mk}-1 non-zero matrices. As both conditions are bilinear, it suffices to consider normalized information vectors and may normalize the columns in the matrix X𝑋X, reducing the total number of matrices by no more than the factor (q−1)msuperscript𝑞1𝑚(q-1)^{m}. Sorting the columns of the matrix gives an additional reduction by a factor of at most m!𝑚m!. Hence using this approach, one has to test at least

qm​k−1m!​(q−1)msuperscript𝑞𝑚𝑘1𝑚superscript𝑞1𝑚\frac{q^{mk}-1}{m!(q-1)^{m}} (5)

matrices in order to show that no extension exists. If one is interested in all possible extension, an exhaustive search is necessary, too. Nonetheless, exhaustive search might be feasible to find an extension if the dimension k𝑘k of the code is small or if many extensions exist.

III-B Extending binary codes by one position

For binary codes, condition (2) can be re-written as

∑i=1kvi​xi=1for all 𝒗∈𝒥d.superscriptsubscript𝑖1𝑘subscript𝑣𝑖subscript𝑥𝑖1for all 𝒗∈𝒥d.\sum_{i=1}^{k}v_{i}x_{i}=1\quad\text{for all $\mathchoice{\mbox{\boldmath{$\displaystyle v$}}}{\mbox{\boldmath{$\textstyle v$}}}{\mbox{\boldmath{$\scriptstyle v$}}}{\mbox{\boldmath{$\scriptscriptstyle v$}}}\in{\cal J}_{d}$.} (6)

The possible extensions of the code correspond to the set of solutions of the inhomogeneous system of linear equations (6). The complexity of computing the solutions if one exists is no longer exponential as in (5), but only polynomial. Moreover, it suffices to compute a subset 𝒥d′subscriptsuperscript𝒥′𝑑{\cal J}^{\prime}_{d} of the information vectors of the minimum weight codewords such that the linear spans of 𝒥dsubscript𝒥𝑑{\cal J}_{d} and 𝒥d′subscriptsuperscript𝒥′𝑑{\cal J}^{\prime}_{d} coincide.

III-E Further remarks

For linear binary codes we have seen that sometimes it is sufficient to compute only a subset of the minimum weight codewords. In general, one can use a subset of 𝒥dsubscript𝒥𝑑{\cal J}_{d} to test whether a code can be extended and compute a set of candidates for the extension using the ideal J𝐽J of eq. (8) or eq. (9). In many cases, the resulting set of candidates is rather small, so that one can perform an exhaustive search among them. Similar, a double extension of a code C𝐶C to a code C′′=[n+2,k,d+2]qsuperscript𝐶′′subscript𝑛2𝑘𝑑2𝑞C^{\prime\prime}=[n+2,k,d+2]_{q} can be found using the solutions for the single extension to C′=[n+1,k,d+1]qsuperscript𝐶′subscript𝑛1𝑘𝑑1𝑞C^{\prime}=[n+1,k,d+1]_{q}.

Kohnert [12, 11] has proposed to compute extensions using integer linear programming by reformulating (1) as hitting-set problem. The ground set of the hitting-set problem is the set of all normalized non-zero vectors that can be appended to the generator matrix, so its size grows exponentially in the dimension of the code.

With some few exceptions, e. g., for the codes [89,11,54]5subscript8911545[89,11,54]_{5}, [93,11,57]5subscript9311575[93,11,57]_{5}, [76,8,53]7subscript768537[76,8,53]_{7}, [45,8,30]9subscript458309[45,8,30]_{9}, computing all solutions via a Gröbner basis is even faster than finding a single solution by exhaustive search.

Table II contains some binary and ternary codes whose minimum distance can be increased by appending two columns to the generator matrix. For these codes, the Gröbner basis approach is quite fast, but unfortunately, this is not always true.

There is a ternary code C=[178,23,81]3𝐶subscript17823813C=[178,23,81]_{3} with 80 words of weight 81 that can be extended to a code C′=[179,23,82]3superscript𝐶′subscript17923823C^{\prime}=[179,23,82]_{3}. Computing a Gröbner basis took about 79 hours on an AMD Opteron 252 (clock speed 2.6 GHz), using about 16 GB of memory. Using exhaustive search, a solution was found in 189.730 seconds, while the projected total running time for the complete exhaustive search is more than 250 hours.

Furthermore, there is a quasicyclic code C=[140,19,73]4𝐶subscript14019734C=[140,19,73]_{4} with 840 words of weight 737373 that can be extended to a code C′=[142,19,74]superscript𝐶′1421974C^{\prime}=[142,19,74]. Using exhaustive search, a solution was found after 4.36 hours on an AMD Opteron 250 (clock speed 2.4 GHz). After 35.75 days of CPU time, 654 solutions have been found while the projected total running time for the exhaustive search is 1011superscript101110^{11} years. Computing a Gröbner basis for the ideal J𝐽J of this code seems to be infeasible.

It turns out that the codes with parameters [66,22,22]3subscript6622223[66,22,22]_{3}, [67,23,22]3subscript6723223[67,23,22]_{3}, [78,11,47]5subscript7811475[78,11,47]_{5}, [51,6,37]7subscript516377[51,6,37]_{7}, and [76,8,53]7subscript768537[76,8,53]_{7} are doubly extendible. The codes [172,17,70]2subscript17217702[172,17,70]_{2} and [173,18,70]2subscript17318702[173,18,70]_{2} in Table II can be extended in two steps to codes [175,17,72]2subscript17517722[175,17,72]_{2} and [176,18,72]2subscript17618722[176,18,72]_{2}. The codes [119,7,75]3subscript1197753[119,7,75]_{3} and [85,9,51]3subscript859513[85,9,51]_{3} can even be extended to codes [123,7,78]3subscript1237783[123,7,78]_{3} and [89,9,54]3subscript899543[89,9,54]_{3}.
