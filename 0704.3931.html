The Complexity of Model Checking Higher-Order Fixpoint Logic

Roland Axelsson\rsupera \lsuperaDepartment of Computer Science, University of Munich, Germany Roland.Axelsson@ifi.lmu.de ,  Martin Lange\rsuperb \lsuperbDepartment of Computer Science, University of Aarhus, Denmark Martin.Lange@ifi.lmu.de  and  Rafał Somla\rsuperc \lsupercIT Department, Uppsala University, Sweden Rafal.Somla@it.uu.se
Abstract.

Higher-Order Fixpoint Logic (HFLHFL\mathrm{HFL}) is a hybrid of the simply typed λ𝜆\lambda-calculus and the modal μ𝜇\mu-calculus. This makes it a highly expressive temporal logic that is capable of expressing various interesting correctness properties of programs that are not expressible in the modal μ𝜇\mu-calculus.

This paper provides complexity results for its model checking problem. In particular, we consider those fragments of HFLHFL\mathrm{HFL} that are built by using only types of bounded order k𝑘k and arity m𝑚m. We establish k𝑘k-fold exponential time completeness for model checking each such fragment. For the upper bound we use fixpoint elimination to obtain reachability games that are singly-exponential in the size of the formula and k𝑘k-fold exponential in the size of the underlying transition system. These games can be solved in deterministic linear time. As a simple consequence, we obtain an exponential time upper bound on the expression complexity of each such fragment.

The lower bound is established by a reduction from the word problem for alternating (k−1)𝑘1(k-1)-fold exponential space bounded Turing Machines. Since there are fixed machines of that type whose word problems are already hard with respect to k𝑘k-fold exponential time, we obtain, as a corollary, k𝑘k-fold exponential time completeness for the data complexity of our fragments of HFLHFL\mathrm{HFL}, provided m𝑚m exceeds 3. This also yields a hierarchy result in expressive power.

1. Introduction

Temporal logics are well-established tools for the specification of correctness properties and their verification in hard- and software design processes. One of the most famous temporal logics is Kozen’s modal μ𝜇\mu-calculus ℒμsubscriptℒ𝜇\mathcal{L}_{\mu} [Kozen83] which extends multi-modal logic with extremal fixpoint quantifiers. ℒμsubscriptℒ𝜇\mathcal{L}_{\mu} subsumes many other temporal logics like PDL [Fischer79] as well as CTL∗ [Emerson:1986:SNN], and with it CTL [EmersonHalpern85] and LTL [Pnueli:1977]. It also has connections to other formalisms like description logics for example.

ℒμsubscriptℒ𝜇\mathcal{L}_{\mu} is equi-expressive to the bisimulation-invariant fragment of Monadic Second Order Logic over trees or graphs [focs91*368, CONCUR::JaninW1996]. Hence, properties expressed by formulas of the modal μ𝜇\mu-calculus are only regular. There are, however, many interesting correctness properties of programs that are not regular. Examples include uniform inevitability [Emerson:1987:UIT] which states that a certain event occurs globally at the same time in all possible runs of the system; counting properties like “at any point in a run of a protocol there have never been more send- than receive-actions”; formulas saying that an unbounded number of data does not lose its order during a transmission process; or properties making structural assertions about their models like being bisimilar to a linear time model.

When program verification was introduced to computer science, programs as well as their correctness properties were mainly specified in temporal logics. Hence, verification meant to check formulas of the form φ→ψ→𝜑𝜓\varphi\to\psi for validity, or equally formulas of the form φ∧ψ𝜑𝜓\varphi\wedge\psi for satisfiability. An intrinsic problem for this approach and non-regular properties is undecidability. Note that the intersection problem for context-free languages is already undecidable [barhillel-etal:1961a].

One of the earliest attempts at verifying non-regular properties of programs was Non-Regular PDL [JCSS::HarelPS1983] which enriches ordinary PDL by context-free programs. Non-Regular PDL is highly undecidable, hence, the logic did not receive much attention for program verification purposes. Its model checking problem, however, remains decidable on finite transition systems – it is even in P [lange-mcpdl].

Another example is Fixpoint Logic with Chop, FLC, [Mueller-Olm:1999:MFL] which extends ℒμsubscriptℒ𝜇\mathcal{L}_{\mu} with a sequential composition operator. It is capable of expressing many non-regular – and even non-context-free – properties, and its model checking problem on finite transition systems is decidable in deterministic exponential time [ls-fossacs02]. It also properly subsumes Non-Regular PDL [langesomla-ipl06].

In order to achieve non-regular effects in FLC, the original ℒμsubscriptℒ𝜇\mathcal{L}_{\mu} semantics is lifted to a function from sets of states to sets of states. This idea has been followed consequently in the introduction of Higher-Order Fixpoint Logic, HFLHFL\mathrm{HFL}, [viswanathan:hfl] which incorporates a simply typed λ𝜆\lambda-calculus into the modal μ𝜇\mu-calculus. This gives it even more expressive power than FLC. HFLHFL\mathrm{HFL} is, for example, capable of expressing assume-guarantee-properties. Still, HFLHFL\mathrm{HFL}’s model checking problem on finite transition systems remains decidable. This has been stated in its introductory work [viswanathan:hfl]. It is also known that model checking HFLHFL\mathrm{HFL} is non-elementary with the following complexity bounds [langesomla-mfcs05].

  • •

    When restricted to function types of order k𝑘k, the model checking problem for this fragment is hard for deterministic (k−3)𝑘3(k-3)-fold exponential space and included in determinisitic (k+1)𝑘1(k+1)-fold exponential time. It is not made explicit, though, that the arity of types needs to be fixed for that.

  • •

    The model checking problem is non-elementary on fixed (and very small) structures already. However, unbounded type orders are needed for this result.

Our aim is to close this apparent gap and to provide an analysis of the model checking problem for HFLHFL\mathrm{HFL} and, thus, the problem of automatically verifying non-regular properties on finite transition systems.

We start in Sect. 2 by recalling the logic and giving a few examples of HFLHFL\mathrm{HFL}-expressible properties. Sect. 3 contains a reduction from HFLHFL\mathrm{HFL}’s model checking problem to the problem of solving (rather large) reachability games. This improves the upper bound mentioned above: these games can be solved in k𝑘k-fold exponential time when type orders are bounded by k𝑘k and arities are fixed.

Sect. LABEL:sec:lower presents a reduction from the word problem for alternating space-bounded Turing Machines to HFLHFL\mathrm{HFL}’s model checking problem. This improves on the lower bounds mentioned above in two ways. For the fragment of type orders restricted to k𝑘k we can match the new upper bound and establish completeness for the class of k𝑘k-fold deterministic exponential time. A slight modification produces formulas that are independent of the input word to the Turing Machine. Hence, we get a result on the data complexity of HFLHFL\mathrm{HFL} as a simple corollary. This, in turn, yields a hierarchy result on expressive power within HFLHFL\mathrm{HFL}.

A non-elementary lower complexity bound on the problem of a logic that incorporates the simply typed λ𝜆\lambda-calculus is of course reminiscent of Statman’s result which states that the normalisation problem in the simply typed λ𝜆\lambda-calculus is non-elementary [statman79tcsb]. But this is rather related to the equivalence problem for HFLHFL\mathrm{HFL} which is known to be highly undecidable [JCSS::HarelPS1983, langesomla-ipl06, viswanathan:hfl]. Since HFLHFL\mathrm{HFL} is a branching time logic there is probably no simple reduction from the equivalence problem to the model checking problem. Hence, the lower bounds presented here do not necessarily follow from Statman’s result.

Furthermore, Statman’s result is of course irrelevant for the upper bounds presented here. There is some work on upper bounds for the number of β𝛽\beta-reduction steps in the simply typed λ𝜆\lambda-calculus, c.f. [Schwichtenberg91]. However, this is not good enough to obtain the upper bounds we are after, c.f. Sect. 3. It also does not deal with the propositional, modal and fixpoint parts of HFLHFL\mathrm{HFL} formulas.

3. The Upper Bound

We will take two steps in order to obtain a k𝑘kExpTime upper bound on the model checking problem for HFLk,msuperscriptHFL𝑘𝑚\mathrm{HFL}^{k,m} for every m∈ℕ𝑚ℕm\in\mathbb{N}. First we eliminate fixpoint constructs from the formula w.r.t. the underlying transition system. This results in a possibly k𝑘k-fold exponentially larger modal formula with λ𝜆\lambda-abstractions and function applications. We then reduce the model checking problem for such formulas to the reachability game problem in graphs of roughly the same size.

The combination of the elimination step and the reduction step is necessary to achieve the k𝑘kExpTime upper bound. It would be easy to eliminate the λ𝜆\lambda-calculus part from a fixpoint-free formula using β𝛽\beta-reduction. However, the best known upper bounds on the number of reduction steps in the simply typed λ𝜆\lambda-calculus are approximately of the order 𝟐k+1O​(n)subscriptsuperscript2𝑂𝑛𝑘1\mathbf{2}^{O(n)}_{k+1} [Schwichtenberg91] which would only yield a (2​k+1)​ExpTime2𝑘1ExpTime(2k+1)\textsc{ExpTime} upper bound.

The reason for avoiding the additional k+1𝑘1k+1 exponents is that β𝛽\beta-reduction is a purely syntactical procedure. We incorporate semantics into these reachability games by evaluating λ𝜆\lambda-bound variables to real functions of finite domain and co-domain rather than unwinding the entire syntactical definition of that function as a program in the simply typed λ𝜆\lambda-calculus. Note that such a function can be represented by more than one λ𝜆\lambda-term. Whereas equivalence of fixpoint-free HFLHFL\mathrm{HFL} formulas is difficult to decide – in fact, it is undecidable in general and might require β𝛽\beta-reduction on a fixed transition system – it is easy to decide for unique semantical representations of these functions.
