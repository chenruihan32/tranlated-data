Abstract

We present an algorithm for systematic encoding of Hermitian codes. For a Hermitian code defined over G​F​(q2)𝐺𝐹superscript𝑞2GF(q^{2}), the proposed algorithm achieves a run time complexity of O​(q2)𝑂superscript𝑞2O(q^{2}) and is suitable for VLSI implementation. The encoder architecture uses as main blocks q𝑞q varying-rate Reed-Solomon encoders and achieves a space complexity of O​(q2)𝑂superscript𝑞2O(q^{2}) in terms of finite field multipliers and memory elements.

Index Terms:
Hermitian Codes, Systematic Encoding, Vandermonde Metrices.

I Introduction

Algebraic-Geometric (AG) codes [1] offer desirable properties such as large code lengths over small finite fields, the potential to find a large selection of codes and good error-correction at high code rates [2]. In recent years, an important class of one-point AG codes, called Hermitian codes, has been frequently discussed [3]-[6].

For a Hermitian code defined over G​F​(q2)𝐺𝐹superscript𝑞2GF(q^{2}), a brute-force way to design an encoder is to multiply the information vector by a generator matrix. The space complexity of a serial-in serial-out architecture for this systematic encoder is O​(q5)𝑂superscript𝑞5O(q^{5}) in terms of finite field multipliers and O​(q3)𝑂superscript𝑞3O(q^{3}) in terms of memory elements. The encoder requires 2​n2𝑛2n clock cycles to generate a codeword of length n𝑛n, thus, the latency is n𝑛n.

By considering a Hermitian code as a superposition of several generalized Reed Solomon (RS) codes, an encoding scheme is introduced in [4]. In [5], an encoding algorithm by forming a bivariate information polynomial and evaluating this polynomial at every finite rational point on the Hermitian curve is proposed. However, both such schemes are nonsystematic and involve the evaluation of bivariate polynomials at n𝑛n finite rational points, thus, they may not have efficient hardware architecture for implementations.

A computationally efficient approach for systematic encoding was proposed in [7]. A serial-in serial-out architecture for this approach was proposed in [8]. This architecture requires n𝑛n clock cycles to encode a codeword of length n𝑛n. The space complexity for this architecture is O​(q3)𝑂superscript𝑞3O(q^{3}), both in terms of finite field multipliers and memory elements.

In this paper, we present an algorithm for systematic encoding and syndrome computation of Hermitian codes. We give an outline for the encoder architecture, which uses q𝑞q varying-rate RS encoders as main blocks and requires n2/3superscript𝑛23n^{2/3} clock cycles for encoding a codeword of length n𝑛n. The space complexity of the architecture is O​(q2)𝑂superscript𝑞2O(q^{2}) in terms of both, memory elements and finite field multipliers.

V Final Remarks

A low complexity algorithm for systematic encoding and syndrome computation of Hermitian codes has been presented. The algorithm has a run time complexity of O​(n2/3)𝑂superscript𝑛23O(n^{2/3}) and is suitable for VLSI implementation. We give an outline for the encoder architecture, which uses as main blocks, q𝑞q varying-rate Reed Solomon encoders. The architecture achieves a much lower space complexity in terms of finite field multipliers and memory elements when compared to earlier reported works.
